# Corpus Definition (v1)

## Theme
Enterprise IT governance and cloud architecture documentation.

## Source Type
Official public documentation related to:
- Cloud governance
- Security baselines
- Architecture standards
- Operational procedures
- Identity and access management
- Well-Architected best practices

## Language
English only.

## Inclusion Criteria
- Long-form documentation (minimum ~800 words)
- Clearly structured content (headings, sections)
- Policy, architecture, procedure, or governance focus
- Stable public source

## Exclusion Criteria
- Marketing content
- Blog posts
- News articles
- Highly image-based PDFs
- Duplicate content

## Corpus Size Target
120 documents (+/- 10)

## Versioning
Each document is recorded in manifest_v1.csv.
Each document is hashed (SHA256).
Raw files are stored locally and not committed to Git.
